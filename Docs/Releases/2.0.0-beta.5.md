# UE Project Intelligence 2.0.0-beta.5

Beta.5 closes the contract and workflow gaps recorded in `UEPIRecheck-2026-07-20-beta4.md`.

## Read Consistency

- Live generations retain the latest non-tombstoned fragment for every asset, independent of targeted refresh order.
- `uepi_context(refresh="force")` refreshes every Hard Scope asset atomically and reports requested, refreshed, failed assets, and generation.
- Blueprint input nodes export stable `FKey` identity plus localized display names; gameplay routing resolves and filters the requested key before traversal.
- Focused Blueprint reads preserve Focus, Pins, direct links, query source, and structural sections under projection.
- Payload metadata separates pre-projection pressure from final byte-limit state.

## Service And Recovery

- Status exposes loaded module path, service process identity/start time, loaded/disk source hashes, and restart-required diagnostics.
- Editor build IDs include a compile-specific suffix.
- Recovery markers can be inspected offline, including current and backup SHA-256 fingerprints and a recommended action.
- Exact-session `uepi_recovery_finalize` and `uepi_recovery_rollback` clear or restore a prepared transaction before mutation resumes.

## Runtime And Authorization

- Runtime wait/assert unwrap typed values before comparison.
- Input results distinguish API acceptance, delivery attempt, known handling, and actual Enhanced Input injection.
- Approved BlueprintPure debug functions can be read without exposing arbitrary invocation.
- `agent_objective`, `human_pie`, and default `hybrid` verification keep technical proof separate from user-owned visual acceptance.
- `ReviewEachPlan`, `TrustedSession`, and `TrustedProject` authorization modes preserve immutable planning and all transaction guards while allowing in-policy trusted plans to apply without chat approval.
- Apply returns a complete post-action report with intent, operation outcomes, assets, validation, save, semantic diff, backup/rollback, runtime, and human visual status.

## Compatibility

- Unreal Engine: 5.3.2 release target.
- MCP client: Codex-first stdio profile.
- Existing installations remain in `ReviewEachPlan` until the user explicitly enables a trusted mode in Project Settings.
