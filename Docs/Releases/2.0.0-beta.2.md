# UEPI 2.0.0-beta.2 Verification

Date: 2026-07-13

## Scope

This Beta raises the practical Agent transaction budget while strengthening immutable Preview enforcement, one-approval continuation, all-target compensation, and Agent-visible failure evidence.

## Verified

- Default guarded limits are 96 operations and 12 affected assets; Preview blocks over-budget plans before approval, while configurable hard caps remain 256 operations and 64 assets.
- UEPIBlank completed an eight-operation/four-asset create/configure/save/refresh/diff transaction, exact-read all four assets, explicitly rolled back all four, and returned restart-stable tombstones.
- A second four-asset transaction injected failure after operation four. The journal entered `failed_rolled_back`, `saved=false`, all four package files remained absent, and current reads distinguished the two compensated targets from the two operations that never executed.
- MCP error envelopes retain non-empty structured failure results, keeping `atomicity_restored` and compensation evidence visible to the Agent.
- Python package compilation and `Tools/test_snapshot_mcp_v2.py` pass.
- The extracted Source candidate passes isolated UE5.3.2 `RunUAT BuildPlugin -TargetPlatforms=Win64 -Rocket`.
- Both archives use one `UEProjectIntelligence/` root, report `2.0.0-beta.2`, and exclude generated/private state. The Source archive contains no binaries; the Win64 archive contains the isolated-build DLL.
- A fresh project install from the Win64 archive completed `UEPIIndex` with 1896 assets, 4759 relations, and zero diagnostics, then passed online Doctor 17/17 with 64 operations and 96/12 limits.

## Golden Evidence

- Multi-asset success, rollback, and injected failure: `Docs/RealMachineTests/2026-07-13-atomic-transactions.md`.
- Beta.1 P0 Golden and multi-project records remain valid and are linked from `Docs/Releases/2.0.0-beta.1.md`.
