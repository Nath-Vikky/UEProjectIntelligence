# UEProjectIntelligence 2.0.0-beta.4

This beta addresses the P0/P1/P2 findings in `UEPIRecheck-2026-07-17-beta3.md` and moves UEPI from isolated asset operations toward a continuous gameplay-development loop.

## Highlights

- `gameplay_input_to_effect` resolves input ownership and follows direct cross-Blueprint calls to terminal gameplay effects while reporting duplicate paths and unresolved dynamic dispatch.
- Blueprint reads preserve Pin containers and raw links; exact node reads prioritize the requested node, complete Pins, and direct one-hop evidence.
- Preview and Apply share Editor-resolved graph/node/pin identities. Failed operations retain target evidence and distinguish not-modified, compensated, pending-recovery, and failed-recovery states.
- Asset Registry/module readiness and baseline compile block unsafe edits. File restoration is separated from deferred Package reload outside the socket request stack.
- Standalone Runtime Preview/Approve provides a restricted PIE ticket without an unrelated asset write. Runtime supports explicit input delivery, typed Enhanced InputAction injection, delay, waits, assertions, and PIE start waiting.
- Targeted refresh publishes an asset tombstone before its replacement fragment, preventing old graph relations from leaking into the new generation.
- Hard Scope remains cache-backed, metadata is not selected as a pagination root, and force refresh performs bounded request/wait/retry.
