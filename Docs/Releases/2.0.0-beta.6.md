# UE Project Intelligence 2.0.0-beta.6

Beta.6 addresses the correctness, recovery, Runtime, and performance findings in `UEPIRecheck-2026-07-20-beta5.md`.

## Input Correctness

- `gameplay_input_to_effect` accepts exact `input_key` and `excluded_input_keys` and uses `key=...` ranking hints only as a secondary structured signal.
- Digit aliases normalize to Unreal stable FKeys such as `3 -> Three`; `LeftAlt` casing remains stable.
- English and Chinese exclusions are honored. Multiple positive keys and unknown keys return blocking diagnostics with empty selected inputs, paths, terminal effects, matches, relations, and Runtime next actions.

## Recovery And Runtime

- Old relative recovery paths resolve under the bound project; new journals persist absolute normalized package and backup filenames.
- `uepi_recovery_discard` can retire an obsolete prepared marker only after explicit user acknowledgement and matching service/Editor current-file fingerprints.
- Runtime key delivery distinguishes API return handling from unproven Blueprint binding execution.
- Normal Runtime `read(field=...)` returns a bounded `uepi.runtime-read-projection.v1` child response.

## Read Performance

- Animation Context uses a bounded summary path instead of constructing a full animation response before field projection.
- SQLite cache v2.2 stores per-entity observation timestamps so hot freshness checks avoid scanning every immutable fragment.
- Focused Blueprint reads remove duplicated heavy metadata while retaining the exact node, Pins, direct links, semantic summary, and query source.
- Truncation metadata reports final projection completeness separately from pre-projection pressure.
