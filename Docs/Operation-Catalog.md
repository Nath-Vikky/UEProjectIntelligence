# Operation Catalog

`uepi_edit_discover` returns the catalog exported by the exact active Editor session. Its `catalog_hash` is bound into Plan v2 and checked again at Apply. An unknown operation is blocking; Python does not maintain a second allowlist.

## Descriptor Contract

Each descriptor identifies its name, domain, risk, target fields, save requirement, rollback mode, validation mode, and atomic support. The catalog and Plan are versioned and hashed. Clients must not infer support from old documentation; Discover is authoritative.

## Plan Rules

- Use exact object paths and GUID/pin identities read from UEPI.
- Put all dependent operations in one plan and use transaction-local `ref` values.
- Prefer compact Blueprint designs using state, timers, loops, custom events, and helper functions.
- Do not silently replace an unsupported design with a much larger graph.
- Apply is idempotent per transaction and plan hash.
- Plan expiry, catalog changes, session changes, project changes, dirty targets, read-only files, budget excess, or before-fingerprint changes block before mutation.

The machine-readable current contract is generated by `Tools/dump_mcp_contract.py`.
