# LLMNPCDemo vNext Baseline Audit

Date: 2026-07-11

Baseline commit: `102a59f67293a621f41b9b98ac3b312875ae1e37`

Target runtime: Unreal Engine 5.3.2 on Windows, with Codex using the project-local UEPI stdio MCP.

## Provenance

The detailed audit and implementation specification is `DOCX/UEPINEXT.md` in the GasDemo workspace. It consolidates the earlier `NeedUE.md` review, LLMNPCDemo observations, prior UEPI plans, and real-machine tests. The original standalone `NeedUE.md` was not present in the plugin repository when this baseline was frozen, so this record does not invent or paraphrase missing source text.

## Frozen Contract

Run:

```powershell
python -B Tools/dump_mcp_contract.py
```

The generated contract is stored at `Tests/Fixtures/LLMNPCDemo/mcp-contract-v1.json`. It captures the 15 MCP tools, server instructions, and the Editor Bridge read/write capability lists before vNext contract changes.

## Reproducible Baseline

- `Tools/test_snapshot_mcp_v2.py` passes against its isolated Snapshot fixture.
- MCP responses use `uepi.mcp-envelope.v1`.
- The Python edit layer owns a fixed supported-operation table.
- The C++ operation registry exists only as a minimal skeleton and is not the sole capability source.
- Read capabilities exist in the Bridge but are not exposed as dedicated Codex tools.

## Frozen Failures

The machine-readable baseline is `Tests/Fixtures/LLMNPCDemo/known-failures-v1.json`.

The five required failures are:

1. A project may be selected through fuzzy session matching instead of an exact local project binding.
2. Context scope is echoed but does not act as a hard result filter.
3. Unknown edit operations are warnings and Preview may still advertise Apply.
4. Apply reports `saved=false`; touched package persistence is not implemented.
5. Generic validation attempts to load every affected target as a Blueprint.

## Fixture Availability

The LLMNPCDemo project itself is not inside this repository. Asset fixtures for `ABP_Manny1`, `Waving`, `BP_LLMNPC_Manny`, and the `LLMNPCActionLayer` plugin must be captured from that real project during its next UE5.3.2 test session. Until then, the contract and known-failure fixtures are authoritative for non-Editor regression tests; missing binary project fixtures are explicitly not treated as passing evidence.
